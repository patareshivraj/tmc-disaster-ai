# Gemini 2.5 Flash Copilot Integration

## 1. Overview
The TMC Disaster Management AI Platform's Generative Copilot layer has been officially migrated from Groq (Llama 3) to **Google Gemini 2.5 Flash** to provide more reliable tool-calling and deeper contextual reasoning during municipal crises.

## 2. Integration Architecture
To maintain absolute stability and zero downtime during the migration, the Gemini API is integrated using the **OpenAI Compatibility Layer**.

By utilizing `generativelanguage.googleapis.com/v1beta/openai/`, the system retains its entire pre-existing agentic reasoning framework, strictly adhering to the legacy OpenAI SDK structures. 

*   **SDK Used:** `openai` Python SDK.
*   **Base URL:** `https://generativelanguage.googleapis.com/v1beta/openai/`
*   **Model ID:** `gemini-2.5-flash`

## 3. Implementation Details
The integration is isolated within the `LLMFactory` and `GeminiProvider` classes located in `ai_engine/llm/`.

### `GeminiProvider`
This class implements the standard `LLMProvider` interface. It handles:
1. Instantiating the OpenAI client pointing to the Gemini endpoints.
2. Managing the mapping of Gemini tool definitions (JSON Schemas) identical to OpenAI format.
3. Handling `tool_calls` responses from the Gemini API and successfully executing the underlying deterministic AI engines (Flood, Resource, Building, Universal).

## 4. Configuration
The LLM Provider is controlled entirely via Environment Variables (`.env`).
```env
LLM_PROVIDER=gemini
GEMINI_API_KEY=your_gemini_api_key_here
```

## 5. Security and Logging
All interactions, tool calls, and responses generated by Gemini are securely logged asynchronously in the `dmd_ai_prediction_log` and `dmd_chatbot_log` MySQL tables via the `ai_monitoring` sidecar, guaranteeing 100% auditability of the AI's reasoning.
