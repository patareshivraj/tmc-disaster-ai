# Recommendation Data Contract

## Objective
This contract explicitly defines the payload schema required by the Phase 11 Recommendation Engine. It standardizes the outputs from the five distinct underlying AI models into a single, cohesive ingestion matrix.

## Ingestion Schema

```json
{
  "ward": "string",
  "flood_probability": "float (0-100)",
  "ward_risk_score": "float (0-100)",
  "resource_shortage_score": "float (0-100)",
  "building_risk_score": "float (0-100)",
  "forecast_incidents": "integer",
  "forecast_severity_critical_pct": "float (0-100)"
}
```

### Definitions:
*   `ward`: Name of the TMC Ward (e.g., "Diva").
*   `flood_probability`: The direct `%` probability of flooding outputted by the **Flood Prediction AI**.
*   `ward_risk_score`: The comprehensive 0-100 hazard score from the **Ward Risk AI**.
*   `resource_shortage_score`: The mathematical gap-score generated by the **Resource Recommendation AI**.
*   `building_risk_score`: The aggregate structural risk assessment outputted by the **Building Advisor AI**.
*   `forecast_incidents`: Expected incident volume within a 7-30 day window generated by the **Incident Forecast AI**.
*   `forecast_severity_critical_pct`: The percentage of future incidents predicted to be "Critical" by the **Incident Forecast AI**.

## Output Schema
The Recommendation Engine guarantees the following output structure:

```json
{
  "ward": "string",
  "combined_risk_score": "float",
  "priority_level": "string",
  "escalation_level": "string",
  "recommendations": [
    {
      "action": "string",
      "priority": "integer",
      "confidence": "float",
      "reason": "string"
    }
  ]
}
```
