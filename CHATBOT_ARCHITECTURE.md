# AI Chatbot Intelligence Layer Architecture

## Overview
The Phase 12 AI Chatbot serves as the single unified interface to the TMC Disaster Management AI Platform. It abstracts away the complexity of the underlying 6 AI modules (Flood, Ward, Resource, Building, Forecast, Recommendation) and replaces dashboards with natural language Q&A.

This architecture is currently an **Intelligent Multi-Agent Orchestrator** running entirely locally without external LLM dependencies, acting as a structured semantic router.

## 1. Intent Engine (`intent_engine.py`)
Responsible for parsing the officer's natural language string into a structured computational intent.
*   Uses semantic keyword matching and regex patterns to classify queries into 10 distinct intents: `Flood`, `Ward Risk`, `Resource`, `Building`, `Forecast`, `Recommendation`, `Dashboard Summary`, `City-Wide`, `Emergency`, `General`.
*   Extracts specific entities (e.g., Ward Names) from the query.

## 2. AI Orchestrator (`chatbot_orchestrator.py`)
The "brain" of the Chatbot. Once the intent is known, the Orchestrator determines exactly which sub-AI modules must be queried.
*   *If Intent == Resource:* Calls `ResourceRecommendationEngine` + `RecommendationEngine`.
*   *If Intent == Emergency / Recommendation:* Calls *All AIs* to build a comprehensive risk matrix.
*   Passes extracted entities (like a specific Ward) downstream to the models.

## 3. Response Composer (`response_builder.py`)
Transforms the raw JSON matrices and float outputs from the sub-AIs into human-readable, officer-grade text.
*   Removes duplicates.
*   Prioritizes critical actions at the top.
*   Ensures Explainability requirements are strictly met by injecting the `reasons` generated by the Phase 11 engine directly into the chat response.

## 4. Main Chatbot Engine (`chatbot_engine.py`)
The entry point. Exposes `answer_question(query)` which chains the pipeline:
`Query` → `detect_intent()` → `route_query()` → `compose_response()` → `Officer Output`.

## Edge Case Handling
*   **Missing Entities:** Fallback to city-wide averages or ask for clarification.
*   **Unknown Intents:** Defers to the Recommendation Engine to provide a standard daily briefing.
*   **Sparse Data:** The Response Composer flags low-confidence or missing outputs gracefully without crashing.
